### 3. Integer Type Default Value

**Status:** ⚠️ Potential confusion  
**Location:** Multiple places  

The documentation states:
- README line 399: `"integerType": "Int32", // Optional. Default="Int32". The .NET type for OpenAPI integers without a format. Possible values: "Int32", "Int64"`
- README line 152: `--integer-type int The .NET type to use for OpenAPI integer types without a format specifier. Common values: 'int' (default), 'long'`

The CLI uses lowercase `int`/`long` while the .refitter format uses `Int32`/`Int64`. This is correct (IntegerType enum handles both) but may confuse users.

**Action:** Clarify that both formats are valid, or standardize the documentation to use one format consistently.
